Question: Un test trop lie à l'implémentation risque de...
A. Casser lors d'un refactoring sans changement de comportement.
B. Rendre le code automatiquement plus rapide.
C. Supprimer le besoin d'objets.
D. Remplacer UML.
Answer: Casser lors d'un refactoring sans changement de comportement.
Explanation: Un test doit protéger le comportement, pas reproduire chaque detail interne.
---
Question: Un fake repository sert a...
A. Tester sans utiliser une vraie basé de données.
B. Cacher une erreur de compilation.
C. Remplacer tous les tests unitaires.
D. Forcer l'héritage.
Answer: Tester sans utiliser une vraie basé de données.
Explanation: Un fake fournit une implémentation simple et contrôlée pour le test.
