# Skills

Organisation actuelle :

- `skills/poo/` pour les skills liées au corpus POO
- chaque skill vit dans son propre dossier
- chaque dossier garde le `SKILL.md` et ses fichiers d'accompagnement éventuels

Quand un nouveau corpus arrive, créer un nouveau dossier de premier niveau, par exemple :

- `skills/java/`
- `skills/system-design/`
- `skills/data/`

Le but est de séparer les skills par domaine de contenu, pas par type de fichier.
