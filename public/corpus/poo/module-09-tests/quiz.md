Question: Un test trop lie à l'implémentation risque de...
A. Casser lors d'un refactoring sans changement de comportement.
B. Rendre le code automatiquement plus rapide.
C. Supprimer le besoin d'objets.
D. Remplacer UML.
Answer: Casser lors d'un refactoring sans changement de comportement.
Explanation: Un test doit protéger le comportement, pas reproduire chaque détail interne.
---
Question: Un fake repository sert a...
A. Tester sans utiliser une vraie base de données.
B. Cacher une erreur de compilation.
C. Remplacer tous les tests unitaires.
D. Forcer l'héritage.
Answer: Tester sans utiliser une vraie base de données.
Explanation: Un fake fournit une implémentation simple et contrôlée pour le test.
---
Question: Que doit vérifier en priorité un test orienté objet ?
A. Le nom des variables privées.
B. Le nombre exact de lignes.
C. Le comportement observable de l’objet.
D. La présence de tous les getters.
Answer: Le comportement observable de l’objet.
Explanation: Le test vérifie qu’une réservation impossible est refusée. Il ne vérifie pas l’ordre interne de toutes les méthodes privées.
