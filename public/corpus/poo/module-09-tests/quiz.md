Question: Un test trop lie a l'implementation risque de...
A. Casser lors d'un refactoring sans changement de comportement.
B. Rendre le code automatiquement plus rapide.
C. Supprimer le besoin d'objets.
D. Remplacer UML.
Answer: Casser lors d'un refactoring sans changement de comportement.
Explanation: Un test doit proteger le comportement, pas reproduire chaque detail interne.
---
Question: Un fake repository sert a...
A. Tester sans utiliser une vraie base de donnees.
B. Cacher une erreur de compilation.
C. Remplacer tous les tests unitaires.
D. Forcer l'heritage.
Answer: Tester sans utiliser une vraie base de donnees.
Explanation: Un fake fournit une implementation simple et controlee pour le test.
